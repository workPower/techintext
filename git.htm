~ version controlling system

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. Git is easy to learn and has a tiny footprint with lightning fast performance .

Developer(s) :Junio Hamano & other's Initial release: 7 April 2005 Lastest release: 29 October 2020 Written in: C, Shell, Perl, Tcl, Python Operating system: Linux, macOS., Windows

git has three area's i.e 1) Working area: Initial Stage currently creating, inserting, updating new data files 2) Index area | Staging area:Second Satge | Ready to commit inserted , updated data in file & 3)Local repository:Final Stage | Commit the new changes of file

further details

git init

adding git to the repository i.e version controlling

git log

Eg: git log

get history record of operations done

git add

move the files from working area to staging area

Eg: git add filename ~ single file

Eg: git add *.type(.txt .c .sh .py .java) ~ specific group-files

Eg: git add * ~ all files

git push

Eg: git push origin master

master: branch

origin: alias name for remote repository

git status

Eg: git status

get present status of file's | branch's

git pull

gitpull = gitfetch + gitmerge

gitfetch: checks the changes|updates in remote repository but it will not update in local repository

gitmerge: will merge updated data from remote to local

git clone

get the remote repository (from remote location) to local repository (to local machine)

Eg: git clone URL (https://gihub.com/user/ repository)

git branch

create Eg: git branch branchname

delete Eg: git branch -d branchname

Note:deletion can done only after the commits of branch to another branch or master

force-delete Eg: git branch -D branchname

Note: all the created,updated commit files in that branch will deleted

git checkout ~local

use the move the currentWorking branch to some-other branch

Eg: git checkout branchname

It is also used for reverse stage of files

Eg: git checkout filename ~single_file

Eg: git checkout . ~multiple_files

Note:it can perform before moving files to Staging_area

git reset (3 types) ~local

mixed_reset (default) Eg: git reset HEAD~1 removes recent one commit

soft reset Eg: git reset --soft HEAD~2 moves 2 commit's ~ files to Staging area so that we can commit if we want recent 2 commits

heard_reset Eg: git reset --hard HEAD~3 it will completely removes last 3 commit's ~updated files & not available even in Staging area | Working area

git revert

remote repository

git revert -Note: You can revert any particular commit

git checkout | git reset -Note: you can reset only step-wise
